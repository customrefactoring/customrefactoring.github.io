How Do Developers Customize Refactoring in Practice?


This website contains complementary data to the paper.

Abstract

Background: Refactoring is a common software development practice. Each refactoring type (e.g. an Extract Method or a Move Method) is composed of a core set of code modifications that aim at improving the code structure. However, developers often have to customize such refactorings by including other modifications or discarding core ones to tailor them to their program's needs. These refactorings are called customized refactorings. Unfortunately, there is a lack of empirical evidence on how developers customize typical types of refactoring in their projects. This knowledge gap may misinform practitioners and tool designers, and may even cause a misalignment between the tooling support and the application of refactorings in practice. Thus, our retrospective study investigates how developers customize four common types of refactorings in practice. We analyzed what code modifications developers perform along with which refactoring customizations. We conducted an in-depth analysis of 1,162 refactoring instances performed in 13 software projects. Our results reveal that developers often apply recurring modifications in their customized refactorings that are not covered by Fowler's mechanics (e.g., 80.77% of the instances of Move Method). To make matters worse, several frequent customizations are not fully supported by a popular refactoring tool. Finally, our study also provides recurring customized refactorings that can help: (i) to better understand the refactoring customization needs, and (ii) to further improve tooling support for customized refactoring.
